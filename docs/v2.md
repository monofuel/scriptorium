# v2 plan

Stall detection and retry for coding agents. When an agent finishes a turn without calling `submit_pr`, the orchestrator detects the stall, reminds the agent of the original ticket, and retries.

## Completed

- [x] log parsing
  - Codex: `codex exec --json` for a json stream, parsed with jsony.
  - Claude Code: `--output-format=stream-json` for streaming events.
  - Both harnesses forward useful events (tool use, thinking, text) via `onEvent` callbacks.
- [x] stall detection
  - When a turn ends without `submit_pr`, the orchestrator retries with a continuation prompt reminding the agent of the original ticket.
  - Configurable max attempts; failed tickets are reopened automatically.
- [x] stall + test failure detection
  - Not yet augmented with test output attachment (future work).

### Future work

- coding agent promotions — having managers promote or reassign failing coding agents.
- attach test failure output to stall retry continuation prompts.

## Resolved Issues

### Stale worktree metadata in Docker

Fixed in `addWorktreeWithRecovery` (orchestrator.nim). The function now runs `git worktree prune` before every `git worktree add` call, clearing stale `.git/worktrees/` entries left behind when container `/tmp` paths are cleaned up between runs. A dedicated unit test ("stale worktree metadata is pruned before creating plan worktree") simulates the Docker scenario and verifies recovery.

### Merge queue cannot update plan branch ticket state

Fixed by the same `git worktree prune` fix above. The plan worktree is now reliably created on each tick, so ticket state transitions (in-progress to done) commit successfully after merge.

### Dirty worktree blocks merge

Fixed by auto-committing uncommitted changes in ticket worktrees before merge queue processing and before enqueue.

### Stuck tickets cause infinite retry loops

Fixed by parking tickets after `MaxMergeFailures` consecutive failures. Parked tickets are excluded from `areasNeedingTickets` to prevent re-assignment.

### Docker container runs as non-root

Fixed by adding a `scriptorium` user (uid 1000) in the Dockerfile. claude-code requires non-root for `--dangerously-skip-permissions`.

### nim.cfg host paths don't resolve in container

Fixed by copying `.nimble` and `.nimby` to the container user's home and running `nimby sync -g nimby.lock` in the entrypoint.

### Git identity missing for non-root user

Fixed by adding `git config --global` in the Dockerfile.

### Manager spawns one session per area

Fixed by batching all areas into a single manager session using `manager_tickets_batch.md` with an `{{AREAS_BLOCK}}` placeholder.

### Manager creates tickets for already-implemented code

Fixed at the prompt level: the batch template instructs the agent to check for existing implementations and place completed areas into `tickets/done/`.

### Done tickets not tracked by area suppression

Fixed by adding `PlanTicketsDoneDir` to the `collectActiveTicketAreas` scan loop. Write guards and git add expanded to allow `tickets/done/` writes.

### All agent tiers using Opus

Fixed by configuring `claude-sonnet-4-6` for manager and coding agents, keeping Opus only for the architect.
