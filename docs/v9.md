# v9 plan

Focus: operational reliability improvements.

## Run integration tests in submit_pr pre-check

Currently the `submit_pr` pre-check (`mcp_server.nim`) only runs `make test`
(unit tests) before accepting a PR submission. The merge gate
(`RequiredQualityTargets` in `merge_queue.nim`) runs both `make test` and
`make integration-test`. This gap means the coding agent doesn't learn about
integration test failures until merge time, wasting a full review cycle and
causing the ticket to be reopened.

The fix: have `runWorktreeMakeTest` (or add a new `runWorktreePreCheck`) run
both `make test` and `make integration-test` before accepting the `submit_pr`
call. If either fails, reject the submission so the coding agent can fix the
issue in the same session rather than discovering it after review.

This was observed during v7 execution: ticket 0061 (status v3 enhancements)
passed unit tests but broke an integration test, requiring a full retry cycle.
